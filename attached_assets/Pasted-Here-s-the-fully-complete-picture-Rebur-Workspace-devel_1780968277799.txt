The core rules that never change:

Workspace is freeform вЂ” organize it however your game needs
Scripts follow suffix rules wherever they live вЂ” .server, .client, .shared
.shared scripts only live in Scripts/Shared, never on entities
StarterGui only allows .client scripts, GUI is always client side
Assets/Shared only allows .client scripts, already replicated
Assets/Server only allows .server scripts, never sent to clients
Effects are components on entities, never a container
Data/Session resets, Data/Store persists, always one decision
Teams, Chat, Sound are flat services, no sub containers
Never nest containers inside containers beyond what is shown
Everything is one decision вЂ” the structure always answers where something lives
